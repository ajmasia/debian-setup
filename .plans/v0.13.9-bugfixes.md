### Task 1: Fix label collision in appthemes.sh

- [ ] Rename appthemes-local labels to `_APPTHEMES_ATUIN_LABEL`, `_APPTHEMES_STARSHIP_LABEL`, `_APPTHEMES_ALACRITTY_LABEL` (and any other collisions)

- [ ] Validate: Shell Tools menu shows "Atuin", "Starship" correctly
- [ ] Validate: Software > Terminals shows "Alacritty" correctly
- [ ] Validate: UI and Theming > App Themes shows theme labels correctly
- [ ] Commit: `fix: rename appthemes labels to avoid collision with module globals`

**Bug**: `gnome/appthemes.sh` defines `_ATUIN_LABEL`, `_STARSHIP_LABEL`, `_ALACRITTY_LABEL` which overwrite the globals from `shell/atuin.sh`, `shell/starship.sh`, and `software/alacritty.sh`. The Shell Tools and Terminals menus show "Atuin Theme" instead of "Configure Atuin", etc.

### Task 2: Fix memory percentage in health.sh

- [ ] Replace `free -h` with `free -b` or `free --mega` for awk arithmetic
- [ ] Format the display value separately for human-readable output
- [ ] Validate: Health check shows realistic memory percentage
- [ ] Commit: `fix: use numeric free output for memory percentage in health check`

**Bug**: `free -h` outputs human-readable strings (e.g., "7.6Gi"). Awk attempts arithmetic division on these strings → always shows `(0.0%)`.

### Task 4: Fix spinner leak on Ctrl+C

- [ ] Update INT trap to call `ui::spin_stop` before exiting
- [ ] Ensure `ui::spin_stop` is safe to call when no spinner is running
- [ ] Validate: Ctrl+C during spinner leaves no orphaned processes

**Bug**: The INT trap in the entry point does not call `ui::spin_stop`. If Ctrl+C is pressed while a spinner is running, the background subshell is orphaned.
