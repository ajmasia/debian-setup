# Architecture

## Entry point (`debian-setup`)

- `set -euo pipefail` + trap SIGINT (exit 130)
- Resolves symlinks to get `SCRIPT_DIR`, reads `VERSION`
- When running as root, defines a pass-through `sudo()`
- Sources in strict order: colors > xdg > log > system > gum > ui > apt (lib) > menu > all modules
- Parses CLI flags: `-s`, `-si`, `-sr`, `-l`, `-o <name>`, `-v`, `-h`, `--update`, `--uninstall`
- `-l`/`--list` and `-v`/`--version` run before header/gum (no UI, stdout only)
- Other flags call `gum::check` (auto-installs gum if missing), paint `ui::header`, then route
- Routes to: `menu::main`, `menu::search`, `menu::search_to_install`, `menu::search_to_remove`, `menu::jump`

## Menu system (`src/menu.sh`)

- `menu::main()` — 12 top-level categories via `gum::filter` (height 14), last entry "⚙ Settings"
- `menu::search()` — flat list of all leaf tasks from 28 `_*_TASKS` arrays (listed in `_SEARCH_ARRAYS`)
- `menu::search_to_install()` / `menu::search_to_remove()` — filtered by `check_fn` result
- `menu::list()` — prints all tasks + categories sorted, writes cache to `~/.cache/debian-setup/tasks.txt`
- `menu::jump(query)` — case-insensitive substring match, direct access to task or category
- Shared helpers: `menu::_collect_leaf_tasks(filter)` populates `_COLLECTED_LABELS`/`_COLLECTED_APPLY_FNS`, `menu::_run_choice(label)` routes label to apply_fn
- `_MENU_CATEGORIES` array maps category names to `::run`/`::apply` functions
- Leaf tasks end in `::apply`, sub-aggregators end in `::run`
- Escape/empty string = Back. On main menu = Exit
- All menus include "Back" and "Exit" at the end
- `gum::choose` for small menus (<5 items), `gum::filter` for large (5+ items)
- Module `run()` functions do NOT run checks — show simple labels (strip "Configure ")

## Shell completions

- `completions/debian-setup.bash` and `completions/debian-setup.zsh` — installed via Settings > Completions
- Completions for flags and `-o`/`--option` task names
- Task names read from cache file (`~/.cache/debian-setup/tasks.txt`), not from `debian-setup --list`
- Cache generated by `menu::list()` (on `--list` or on completion install)

## Settings (`src/modules/settings/`)

- Submenu with: System Health, Logs, Completions, About
- Highlighted with "⚙" prefix in main menu
- `health.sh` — machine status + group counters via `_HEALTH_GROUPS` mapping
- `logs.sh` — log file viewer
- `completions.sh` — install/remove bash and zsh completion symlinks separately
- `about.sh` — version, install path, log dir, shell

## Persistent header

Painted once. `_UI_CONTENT_ROW` stores the cursor row (via `\033[6n`, 1-indexed). `ui::clear_content` uses `tput cup $((_UI_CONTENT_ROW - 1)) 0` + `tput ed` (0-indexed). If the terminal has scrolled (after apt-get, sudo, etc.), `_UI_DIRTY=1` triggers a full header repaint. The flag is set in `ui::flush_input`.

## Ctrl+C

`trap 'printf "\n"; exit 130' INT` in entry point. Wrappers propagate exit 130.

## Always-visible modules

System Essentials, Package Managers, Hardware Support, OpenSSH Server, Git, Shell Tools, Development, Dotfiles, UI and Theming, Software, Virtualization, ⚙ Settings — always shown in main menu. No status logging at startup.
